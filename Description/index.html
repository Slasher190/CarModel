API Endpoints Documentation

User Endpoints

  • /cars: Get a list of all cars in the system.

    The response will include information about each car, such as the make, model, year, and price.

  • /cars/dealership/<dealership_id>: Get a list of all cars in a specific dealership.

    The response will include the same information as the /cars endpoint, but it will only include cars that are located in the specified dealership.

  • /cars/<car_id>: Get information about a specific car.

    The response will include all of the information that is returned by the /cars endpoint, as well as any additional information that is specific to the car, such as the VIN number and the dealership that the car is currently located in.

  • /dealerships/with_car/<car_id>: Get a list of all dealerships that have a certain car in stock.

    The response will include the name, address, and phone number of each dealership.

  • /vehicles/owned: Get a list of all vehicles owned by the user.

    The response will include information about each vehicle, such as the make, model, year, and mileage.

  • /dealerships/nearby: Get a list of dealerships within a certain range based on the user's location.

    The response will include the name, address, and phone number of each dealership, as well as the distance between the dealership and the user's location.

  • /deals/on_car/<car_id>: Get a list of all deals available on a certain car.

    The response will include the price of the car, the terms of the deal, and the expiration date of the deal.

  • /deals/from_dealership/<dealership_id>: Get a list of all deals available from a certain dealership.

    The response will include the same information as the /deals/on_car/<car_id> endpoint, but it will only include deals that are offered by the specified dealership.

  • /buy/<car_id>: Buy a car after a deal has been made.

    The request must include the car ID and the deal ID. The response will include the purchase confirmation.

Dealership Endpoints

  • /cars: Get a list of all cars.

    The response will include information about each car, such as the make, model, year, and price.

  • /cars/sold: Get a list of all cars sold by the dealership.

    The response will include the same information as the /cars endpoint.

  • /cars/add: Add a car to the dealership.

    The request must include the car details. No response will be returned.

  • /deals: Get a list of all deals provided by the dealership.

    The response will include information about each deal, such as the car details, price, and terms.

  • /deals/add: Add a deal to the dealership.

    The request must include the deal details. No response will be returned.

  • /vehicles/sold: Get a list of all vehicles the dealership has sold.

    The response will include information about each sold vehicle, such as the car details and the buyer's information.

  • /vehicles/sold/add: Add a new vehicle to the list of sold vehicles after a deal is made.

    The request must include the sold vehicle details. No response will be returned.
